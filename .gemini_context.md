# Gemini Context: ZenBar

This file restores context for developer agents working on the `ZenBar` project.

## Overview
`ZenBar` is a custom Windows desktop status bar (similar to macOS's menu bar) implemented as an Application Desktop Toolbar (AppBar) in C++ (Win32).

## Key Files & Structure
- **[Core/AppBar.h](file:///c:/Users/Yuvi/Documents/VS%20Code/ZenBar/Core/AppBar.h) & [Core/AppBar.cpp](file:///c:/Users/Yuvi/Documents/VS%20Code/ZenBar/Core/AppBar.cpp)**: Manages window lifecycle, messages, AppBar shell registration, display changes, and drawing dispatch.
- **[Core/Renderer.h](file:///c:/Users/Yuvi/Documents/VS%20Code/ZenBar/Core/Renderer.h) & [Core/Renderer.cpp](file:///c:/Users/Yuvi/Documents/VS%20Code/ZenBar/Core/Renderer.cpp)**: Direct2D/GDI rendering layer, handles Mica/Acrylic window blur effects and font management.
- **[Widgets/Widgets.h](file:///c:/Users/Yuvi/Documents/VS%20Code/ZenBar/Widgets/Widgets.h) & [Widgets/Widgets.cpp](file:///c:/Users/Yuvi/Documents/VS%20Code/ZenBar/Widgets/Widgets.cpp)**: Contains all widget logic (Clock, CPU, GPU, Network, Volume, Brightness, Battery). Each widget is represented by a plain struct in `g_widgets[]` containing update and draw function pointers.

## Recent Work
- **Volume Device Switcher**: Fixed vtable alignment issue in private `IPolicyConfig` COM interface by adding `ResetDeviceFormat` to correct method offsets on Windows 10/11. Playback device switching now functions correctly.
- **Fullscreen Auto-Hide & Start Menu Overlay**: Implemented a hybrid focus checking mechanism (`IsFullscreenWindowActive()`) in the `WM_TIMER` callback. Hides the status bar when a fullscreen app (game/browser) is active on the monitor, but immediately restores topmost status bar rendering if the fullscreen app loses focus (e.g. when the Start Menu is launched via the Windows key).
